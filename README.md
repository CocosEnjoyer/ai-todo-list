# 📋 Интеллектуальный менеджер задач с интеграцией LLM

## 1. Описание проекта

Веб-приложение для управления задачами (To-Do List) с интегрированным AI-ассистентом на базе GitHub Models (GPT-4o-mini). Приложение позволяет эффективно управлять задачами, а LLM помогает с приоритизацией, категоризацией, декомпозицией сложных задач и анализом рабочей нагрузки.

### Реализованные функции

| Функция                      | Статус | Описание                                                                           |
| ---------------------------- | ------ | ---------------------------------------------------------------------------------- |
| **1** CRUD задач             | ✅     | Создание, чтение, обновление, удаление задач                                       |
| **2** Фильтрация и поиск     | ✅     | По статусу, приоритету, категории, сроку и тексту (на сервере)                     |
| **3** Умная категоризация    | ✅     | LLM определяет категорию задачи (Работа/Учёба/Дом/Покупки/Здоровье/Финансы/Другое) |
| **4** Декомпозиция задачи    | ✅     | LLM разбивает сложную задачу на подзадачи с возможностью редактирования            |
| **5** Предложение приоритета | ✅     | LLM анализирует срок и описание, предлагает приоритет (HIGH/MEDIUM/LOW)            |
| **6** Сводка нагрузки        | ✅     | LLM генерирует честный анализ рабочей нагрузки с упоминанием важных задач          |

## 3. Установка и запуск

### Требования

- Node.js 18+ (для локального запуска)
- Docker и Docker Compose (для контейнеризации, рекомендую, но можно запустить и без него)
- GitHub аккаунт

### 3.1 Получение GitHub токена

Для работы LLM функций необходим GitHub Personal Access Token. **Это бесплатно!**

1. Перейдите в настройки GitHub: [github.com/settings/tokens](https://github.com/settings/tokens)
2. Нажмите **"Generate new token (classic)"**
3. **Название**: `Task Manager LLM`
4. **Срок действия**: выберите 30-90 дней
5. **Scopes**: **Оставьте всё пустым!** (токен без прав работает с GitHub Models API)
6. Нажмите **"Generate token"**
7. **Скопируйте токен** — он покажется только один раз!

### 3.2 Настройка переменных окружения

Скопируйте пример конфигурации:

```bash
cp backend/.env.example backend/.env
```

Отредактируйте backend/.env:

```env
GITHUB_TOKEN='ваш_токен'
PORT=5000
DATABASE_URL="file:/app/prisma/dev.db"
```

### 3.3 Запуск через Docker (рекомендуется)

```bash
git clone https://github.com/CocosEnjoyer/ai-todo-list

cd ai-todo-list

# Запуск всех сервисов
docker-compose up --build

# Просмотр логов
docker-compose logs -f
```

После запуска:

Нужно зайти на http://localhost:3000

Остановка:

```bash
docker-compose down
```

### 3.5 Локальный запуск (без Docker)

Нужно запустить и Backend и Frontend,

### Запуск Backend:

```bash
cd backend
npm install
npx prisma generate
npx prisma migrate deploy
npm run dev
```

### Запуск Frontend:

```bash
cd frontend
npm install
npm run dev
```

После чего зайти на http://localhost:3000 (либо тот, который указан при запуске фронтенда)

## 4. Архитектурные решения

### 4.1 Разделение ответственности

```text
backend/
├── src/
│ ├── controllers/ # Обработка HTTP запросов
│ ├── routes/ # Маршрутизация (версионирована: /api/v1/...)
│ ├── services/ # Бизнес-логика (LLM промпты, валидация)
│ └── index.ts # Точка входа
├── prisma/
│ └── schema.prisma # Модели данных + миграции
└── Dockerfile

frontend/
├── src/
│ ├── components/ # UI компоненты (CSS Modules)
│ ├── hooks/ # React Query хуки
│ ├── api/ # API клиенты
│ └── types/ # TypeScript типы
└── Dockerfile
```

### 4.2 Ключевые решения

TypeScript: Типизация на фронте и бэкенде, автодополнение
Prisma ORM: Типобезопасные запросы к БД, миграции
SQLite: Не требует отдельного сервера, идеально для тестового
React Query: Кеширование, оптимистичные обновления
GitHub Models: Бесплатный доступ к GPT-4o-mini без привязки карты
Docker Compose: Одна команда для запуска всего проекта
API Versioning: Поддержка /api/v1/...

### 4.3 Обработка ошибок LLM

При некорректном ответе от LLM используются fallback значения:

Приоритет: MEDIUM если LLM вернул неверное значение

Категория: Другое если категория не из списка

Подзадачи: стандартный набор из 4 подзадач

JSON: валидация и очистка от markdown-обёрток

## 5. Известные проблемы и ограничения

1. Длинные слова без переносов могут выходить за пределы карточки задачи
2. Общий внешний вид можно улучшить
3. Подзадачи: нельзя менять порядок (drag-and-drop)
4. Подзадачи: нельзя создать сразу при создании задачи (только после)
5. Бесплатные лимиты GitHub Models: 16K токенов на запрос
6. Rate limiting: ~10-20 запросов в минуту
7. Поиск в SQLite регистрозависимый
8. Компоненты и модули слишком большие, можно декомпозировать
9. Нет пагинации

# 7. Устранение неполадок

## Ошибка: GITHUB_TOKEN not found

Проверьте наличие файла backend/.env

Убедитесь, что в нём есть строка GITHUB*TOKEN='ваш*токен'

## Ошибка: LLM API error: 403

Токен истёк — создайте новый в GitHub Settings

Попробуйте токен без scopes

## Docker: порты уже заняты

# Сменить порты в docker-compose.yml

ports:

- "3001:3000" # вместо 3000:3000
- "5001:5000" # вместо 5000:5000

```bash
  cd backend
  npx prisma migrate dev --name init
  npx prisma generate
```
